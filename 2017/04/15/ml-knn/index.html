KNN笔记

发表于 2017-04-15 | 分类于 机器学习

算法

  1. 给一堆训练的点和对应的类别，不需要训练
  2. 给一个预测的点，计算该点和所有点的距离
  3. 对距离排序
  4. 选择其中前k个距离
  5. 找出前k个距离所对应的点的类别
  6. 数一下这些类别，哪种类别最多，那个要预测的点就归属在那个类别里

优点

  • 因为不需要训练，所以也不需要创建规则，也就是机器学习出来的那个函数。因为规则就是所有训练数据点本身。
  • 可以避免样本数量不平衡的问题，因为无论如何不平衡，预测点只和附近的k个点有关。

缺点

  • 计算复杂度高，数据维度为m，训练数据个数为n，那么就要计算n个m维的距离。
  • 模型依赖于样本，所以受限于样本容量。样本容量太小，特征无法均匀分布会有误差，容量太大，判别代价又太大
  • 数据的每个维度（也就是特征）的权重是一样的，这点和NN比较起来就有差距，因为有些特征并不重要，但也用相同的权重去衡量判别分类，而NN会在学习的过程中调整权重。

优化

  • 针对计算量太大：浓缩训练样本，无需使用全部的训练数据来预测，通过一些算法取训练数据的一部分数据来进行预测即可。
  • 针对特征的权重：在距离公式中人工赋予一些权重
